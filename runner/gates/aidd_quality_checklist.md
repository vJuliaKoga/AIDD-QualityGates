# AIDD品質保証チェックリスト（G2: 人間検証）

## ドキュメント情報
- バージョン: 1.0
- 最終更新: 2026-03-04
- 目的: AIドリブン開発における品質保証ゲートG2の実施基準

## リスクレベル定義

| レベル | 説明 | 重み付けスコア | 影響範囲 |
|--------|------|----------------|----------|
| **S** | クリティカル：プロジェクト全体に重大な影響 | 100 | システム全体の安全性・整合性 |
| **A** | ハイリスク：工程品質に直接影響 | 50 | 工程成果物の信頼性 |
| **B** | ミディアムリスク：後工程での手戻りリスク | 20 | 工程間の整合性 |
| **C** | ローリスク：品質向上・効率化 | 5 | ドキュメント品質 |

## 【全工程共通】チェックリスト

### 1. トレーサビリティ検証 [S]
- [ ] すべてのID（要件ID、設計ID、実装ID）が一意であることを確認
- [ ] 親子関係のリンクが双方向で正しく設定されていることを確認
- [ ] 孤立したID（参照されていない/参照先が存在しない）が存在しないことを確認
- [ ] トレーサビリティマトリクスが完全であることを確認

**検証方法**: tracematrix.jsonとの照合、手動でのサンプリング検証  
**合格基準**: リンク切れ0件、孤立ID 0件

---

### 2. 曖昧性排除検証 [A]
- [ ] 「必要に応じて」「適切に」「可能な限り」等の曖昧語が排除されていることを確認
- [ ] 数値基準が明確（「高速」→「200ms以内」等）に記載されていることを確認
- [ ] 主語・目的語が明確に記載されていることを確認
- [ ] 条件分岐がすべて網羅的に記述されていることを確認

**検証方法**: ambiguityreport.jsonの確認、人間による文章レビュー  
**合格基準**: 曖昧語検出0件、定量基準100%記載

---

### 3. スキーマ準拠検証 [A]
- [ ] YAMLファイルがスキーマ定義に完全準拠していることを確認
- [ ] 必須フィールドがすべて入力されていることを確認
- [ ] データ型が正しいことを確認（文字列/数値/配列等）
- [ ] 列挙値が定義範囲内であることを確認

**検証方法**: schemareport.jsonの確認、バリデーションツールによる検証  
**合格基準**: スキーマエラー0件

---

### 4. AI生成コンテンツの検証 [A]
- [ ] AI生成部分が明示的にマークされていることを確認
- [ ] AI生成コンテンツが人間によってレビュー済みであることを確認
- [ ] AI生成コンテンツが文脈に整合していることを確認
- [ ] AI生成コンテンツに事実誤認・ハルシネーションがないことを確認

**検証方法**: 生成ログの確認、ドメインエキスパートによるレビュー  
**合格基準**: 未レビュー項目0件、事実誤認0件

---

### 5. ドキュメント品質 [C]
- [ ] 誤字脱字がないことを確認
- [ ] 表記揺れ（用語統一）がないことを確認
- [ ] フォーマットが統一されていることを確認
- [ ] バージョン管理情報が正しく記録されていることを確認

**検証方法**: スペルチェック、用語集との照合  
**合格基準**: 重大な誤字脱字0件、用語統一率95%以上

## 付録: チェックリスト自動化連携

本チェックリストはG2ゲートにおける人間検証の証跡として機能します。

**他ゲートとの連携**:
- **G1 (曖昧語チェック)**: ambiguityreport.jsonを参照し、共通チェックリスト2.の検証に活用
- **G3 (Schema検証)**: schemareport.jsonを参照し、共通チェックリスト3.の検証に活用
- **G4 (DeepEval)**: deepevalscores.jsonを参照し、実装フェーズ5.5の検証に活用
- **G5 (トレーサビリティ)**: tracematrix.jsonを参照し、共通チェックリスト1.の検証に活用
- **PF (Promptfoo)**: promptfooresults.jsonを参照し、実装フェーズ5.6の検証に活用

  --g3-report schemareport.json \
  --g4-report deepevalscores.json \
  --g5-report tracematrix.json \
  --pf-report promptfooresults.json \
  --output checklistvalidation.json
