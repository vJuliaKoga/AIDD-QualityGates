# バックアップファイル削除スクリプト使用方法

## 概要

`bulk_rename.py` で作成されたバックアップファイル（`.backup_*`）を一括削除するスクリプトです。

## 使い方

### 1. 確認モード（推奨・最初はこちら）

実際には削除せず、どのファイルが削除されるかを確認します。

```bash
python cleanup_backups.py /path/to/your/directory
```

### 2. 実行モード

実際にバックアップファイルを削除します。

```bash
python cleanup_backups.py /path/to/your/directory --execute
```

## 実行例

```bash
# 現在のディレクトリのバックアップファイルを確認
python cleanup_backups.py .

# 特定のディレクトリのバックアップファイルを削除
python cleanup_backups.py /home/user/project --execute
```

## 表示される情報

- **ファイル数**: 見つかったバックアップファイルの数
- **合計サイズ**: バックアップファイルの合計容量
- **ファイル詳細**: 各ファイルのパス、サイズ、更新日時

## 削除対象ファイル

`.backup_YYYYMMDD_HHMMSS` の形式を持つファイルが削除されます。

例：
- `document.md.backup_20260305_115717`
- `script.py.backup_20260304_143022`

## 除外されるディレクトリ

以下のディレクトリは処理されません：
- `.git`, `.svn` (バージョン管理)
- `node_modules`, `__pycache__` (依存関係)
- `.venv`, `venv` (仮想環境)
- `dist`, `build` (ビルド成果物)

## 安全機能

1. **確認モードがデフォルト**: 実際には削除せず、削除対象を確認できます
2. **実行前確認**: `--execute` 時は確認プロンプトが表示されます
3. **詳細情報表示**: 削除されるファイルの情報（サイズ、日時）を表示

## 注意事項

- **一度削除すると復元できません**
- 必ず最初は確認モードで実行してください
- 重要なバックアップは別途保管してから削除してください

## トラブルシューティング

### エラー: PermissionError

ファイルのアクセス権限がない場合のエラーです。

```bash
chmod +x cleanup_backups.py
```

### バックアップファイルが見つからない

以下を確認してください：
- 正しいディレクトリを指定しているか
- バックアップファイルが `.backup_*` の形式になっているか

## 使用例（完全なワークフロー）

```bash
# 1. まず置換を実行（バックアップが作成される）
python bulk_rename.py /path/to/project --execute

# 2. 置換結果を確認

# 3. 問題なければバックアップを削除（確認モード）
python cleanup_backups.py /path/to/project

# 4. 削除対象を確認してから実行
python cleanup_backups.py /path/to/project --execute
```

## ヒント

### 古いバックアップのみ削除したい場合

このスクリプトは全てのバックアップファイルを削除します。
特定の日付以前のバックアップのみ削除したい場合は、
スクリプトを編集してフィルタ条件を追加してください。

### ディスク容量を確認したい場合

確認モードで実行すると、解放予定のディスク容量が表示されます：

```bash
python cleanup_backups.py . 
# → 「解放予定ディスク容量: XX MB」と表示される
```
